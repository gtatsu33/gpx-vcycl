# Virtual Cycling

iPad Safari（Bluefyブラウザ）で動作するバーチャル室内サイクリングPWA。
GPXルートを読み込み、Sarisスマートトレーナーで勾配を再現しながら仮想的にコースを走るアプリ。

---

## セットアップ

```bash
npm install
npm run dev
```

起動後、ブラウザで `https://localhost:5173` を開く。
自己署名証明書の警告が出た場合は「詳細設定」→「このサイトへ進む」を選択。

---

## HTTPS が必要な理由

Web Bluetooth API は **HTTPS または `localhost`** 上でのみ動作する（ブラウザのセキュリティ要件）。

ローカル開発時は `@vitejs/plugin-basic-ssl` が自己署名証明書を自動生成するため、
`npm run dev` だけで HTTPS が有効になる。

---

## iPad（Bluefy）からのアクセス方法

1. PC と iPad を同一 Wi-Fi ネットワークに接続する
2. PC の IP アドレスを確認する

   ```
   ipconfig   # Windows
   ```

   例: `192.168.1.10`

3. Bluefy ブラウザで `https://192.168.1.10:5173` を開く
4. 自己署名証明書の警告が出たら「詳細設定」→「このサイトへ進む」を選択
5. アドレスバーの共有メニューから「ホーム画面に追加」を選ぶと PWA としてインストールされる

> iOS の設定アプリ → 一般 → VPN とデバイス管理 → 証明書を信頼 でも証明書を信頼できる。

## 動作確認（Phase 4 完了条件）

### 事前準備

1. Settings → 「トレーナー負荷制御」トグルを **ON** にして閉じる
2. トレーナー（FTMS対応）を BLE 接続する

### 確認手順

1. デバイスタブの BLE ログに **「Trainer制御: 制御権取得成功 — 負荷制御が有効です」** と表示されること
2. GPX ルートを選択してライドを開始する
3. 平坦区間 → 上り区間に差し掛かったとき、ペダルが重くなること
4. ライド終了（「⏹ 終了」または完走）後、トレーナーの負荷がゼロに戻ること

### 緊急リセット

ライド中にトレーナーが想定外の高負荷になった場合は、  
ライドコントロールバーの **「⚠ Reset Trainer」** ボタンを押す。  
FTMS Reset コマンドが送信され、トレーナーがアイドル状態に戻る。

### 想定される失敗ケース

| 状況 | 挙動 |
|---|---|
| トレーナーが Control Point 非対応 | BLE ログに「制御ポイント利用不可」と表示。負荷制御なしで接続継続（IBD 読み取りは動作） |
| トレーナー制御トグルが OFF | 勾配コマンドを送信しない。フェーズ3までと同じ挙動 |
| ライド中に BLE 切断 | 勾配送信がスキップされる（コンソールにエラー）。再接続後は自動で再開 |
| 急勾配（±25%超）の GPX | ±25% にクリップして送信（トレーナー保護） |
| コマンド応答タイムアウト（3秒） | エラーをログ出力してスキップ。次の送信タイミングで再試行 |

### パラメータ仕様（FTMS Simulation Parameters）

送信パケット（7 バイト）の構成:

| バイト | 内容 | 単位 | 本アプリの送信値 |
|---|---|---|---|
| 0 | Opcode | — | `0x11` |
| 1–2 | Wind Speed | sint16 LE, 0.001 m/s | 0（無風固定） |
| 3–4 | Grade | sint16 LE, 0.01 % | 現在勾配（±25% クリップ） |
| 5 | Crr | uint8, 0.0001 | Settings の転がり抵抗値 |
| 6 | Wind Resistance Coef | uint8, 0.01 kg/m | `0.5 × 1.225 × CdA` |

送信頻度: **1秒ごと**、または勾配変化が ±1% 以上のとき即時送信。
