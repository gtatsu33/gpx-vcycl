====================================================================
最終更新: 2026-07-11
対象: src/ui/hud.js, index.html #tab-route
関連モックアップ: .claude/mockup-route-hud.html（Artifact確認済み）

--------------------------------------------------------------------
0. スコープ
--------------------------------------------------------------------
本仕様は Route モードの「ライド中（走行中）画面」のレイアウト刷新を
対象とする。以下は変更対象外（現状維持）:

  - 事前ライド画面（ルート選択・GPX読み込み・ライド開始前の
    #pre-ride-panel）は今回の変更範囲に含まない
  - HUDが参照するデータソース・計算ロジック（domain/, ble/）は
    変更しない。表示レイアウトのみの変更

--------------------------------------------------------------------
1. 全体レイアウト（ライド中画面）
--------------------------------------------------------------------
画面を縦2段に分割する。

  ┌─────────────────────────────────────────────┐
  │  route-main（可変高さ、横方向 2 カラム）        │
  │  ┌───────────────────────┬─────────────────┐ │
  │  │                       │  風景写真（上半分）│ │
  │  │   3Dマップ（走行風景） │─────────────────│ │
  │  │   + HUDオーバーレイ     │  2Dマップ（下半分）│ │
  │  │   （左 2/3）            │  （右 1/3）        │ │
  │  └───────────────────────┴─────────────────┘ │
  ├─────────────────────────────────────────────┤
  │  コース全体標高表示（現状のまま、フル幅・固定）  │
  ├─────────────────────────────────────────────┤
  │  ride-controls（一時停止／終了／トレーナーリセット）│
  └─────────────────────────────────────────────┘

  - 左カラム : 右カラム = 2 : 1（flex: 2 / flex: 1）
  - 右カラムは上下に均等分割（各 flex: 1）
  - コース全体標高（#course-elevation-map）はオーバーレイではなく
    独立した帯として画面最下部に常時表示。位置・サイズ・描画ロジック
    （hud.js の #renderCourseElevation）は変更しない

--------------------------------------------------------------------
2. 左カラム（2/3）: 3Dマップ + HUDオーバーレイ
--------------------------------------------------------------------
2-1. ベース
  - 現行の #ele-view（3Dマップ／走行風景表示）をカラム全体まで拡大
  - HUD要素はすべて position:absolute の半透明パネルとして
    3Dマップの上にオーバーレイ表示する（3Dマップの描画・タップ操作を
    妨げないよう pointer-events は必要最小限に絞る）

2-2. HUDパネル共通デザイン
  - 「グラスモーフィズム」調の半透明パネル
    - 背景: 上から下へのわずかなグラデーション
      （rgba(23,32,44,0.72) → rgba(10,14,20,0.72)）
    - backdrop-filter: blur(10px) saturate(140%)
    - 枠線: rgba(255,255,255,0.09) 1px、角丸 14px
    - 影: box-shadow で外側にドロップシャドウ、内側に薄いハイライト
      （inset 0 1px 0 rgba(255,255,255,0.06)）
  - 数値: 白・太字・等幅数字（font-variant-numeric: tabular-nums）
    → 背景の明暗が変化しても視認性を確保する最優先事項
  - 単位: 小さめ・#6080a0（既存 --sub 相当）
  - グループごとにアクセント色の縁取り線で系統を区別
    - 中央上部（リアルタイム系）: 上辺に水色ライン（#38bdf8、既存アクセント色）
    - 左（累積系）: 左辺に青系ライン（#64b5f6）
    - 右（ルート情報）: 右辺にオレンジ系ライン（#ffb454）

2-3. 中央上部パネル：リアルタイム系（2段構成）
  文字ラベルは表示しない。アイコン＋数値＋単位のみで構成する
  （視認性は損なわず、画面占有を最小化するため）。

  上段（大きめ・強調）:
    - パワー   [雷アイコン]   例: "212 W"
    - ケイデンス [回転アイコン] 例: "87 rpm"
    - 心拍     [ハートアイコン] 例: "142 bpm"
    フォントサイズ目安: 数値 1.7rem / 単位 0.68rem / アイコン 19px

  下段（小さめ・中央揃え）:
    - 速度   [速度計アイコン] 例: "32.4 km/h"
    - トルク [トルクレンチアイコン] 例: "38.4 N·m"
    フォントサイズ目安: 数値 1.05rem / 単位 0.56rem / アイコン 13px
    行全体を justify-content:center し、上段より横幅が狭くても
    パネル内で中央に配置する

  選定理由:
    - 上段3項目（パワー／ケイデンス／心拍）はトレーニング強度の
      主指標であり常時大きく見たい
    - 下段2項目（速度／トルク）は補助的に確認する頻度が低いため
      小さめでよい

2-4. 左パネル：累積系
  グループタイトル「累積」を小さく表示（任意、視認の補助）。
  上から順に横並びの行（アイコン＋ラベルテキスト＋値）:

    - 経過時間   [時計アイコン]     例: "28:41"
    - 走行距離   [ルート/距離アイコン] 例: "14.2 km"
                 サブテキストで残り距離を併記（例: "残り 5.8 km"）
    - 累積標高   [山アイコン]       例: "＋184 m"

  左パネルのみラベルテキスト（「経過時間」等）を残す。
  中央上部と異なり出現頻度が低いため、アイコンだけでは判別しにくい
  項目（特に距離・標高）の可読性を優先。

2-5. 右パネル：ルート情報
  グループタイトル「ルート情報」を右寄せで表示。

    - 勾配 [上り坂アイコン]
      現在勾配の値を「バッジ」形式で表示。既存の勾配カラールール
      （hud.js の gradientColor() と同一の閾値・配色）をそのまま踏襲:
        - < 3%  : 緑  #2ed573
        - < 6%  : 黄  #ffd32a
        - < 9%  : 橙  #ee7800
        - < 12% : 赤  #ff0000
        - >=12% : 濃赤茶 #4C2E30
      バッジ背景はその色の半透明（例: rgba(255,211,42,0.18)）、
      文字色は同系色の濃いめトーン
    - 標高 [高度アイコン] 例: "612 m"

2-6. 直近1kmの勾配プロファイル（下部オーバーレイ）
  - 左カラム下端に横長のグラスパネルとしてオーバーレイ表示
  - 内容・描画ロジックは既存 hud.js #renderGradientProfile
    （#gradient-profile への SVG描画）をそのまま踏襲し、
    見た目（背景）のみグラスパネル化する
  - 500m / 1km の距離ラベル、現在地マーカー、勾配色分けは現状維持

--------------------------------------------------------------------
3. 右カラム（1/3）: 風景写真 + 2Dマップ
--------------------------------------------------------------------
3-1. 上半分: 走行風景写真
  - 現行の Mapillary パネル（.mapillary-panel、実写のストリートビュー
    画像）をそのまま移動。写真が無い場合の "no image" 表示、
    クレジット表記（© Mapillary）は現状維持

3-2. 下半分: 2Dマップ
  - 現行の 2Dマップ（#map-inner、Leaflet or Google Maps）をそのまま
    移動。現在地マーカー・ルート線・再センターボタン（#recenter-btn）
    は現状の挙動を維持

--------------------------------------------------------------------
4. 画面最下部: コース全体標高
--------------------------------------------------------------------
  - #course-elevation-map をフル幅・固定高さ 52px で表示（現状維持）
  - オーバーレイではなく独立した帯として常時表示
  - 描画ロジック（#renderCourseElevation, #updateCourseMarker）は
    変更しない

--------------------------------------------------------------------
5. 既存要素IDとの対応（実装時の参照用）
--------------------------------------------------------------------
  データ                  既存ID                  表示位置（新レイアウト）
  ----------------------  ----------------------  --------------------------
  速度 (speedKmh)          #hud-speed               中央上部・下段
  距離 (distanceKm)        #hud-distance            左パネル
  残り距離                 #hud-remaining           左パネル（サブテキスト）
  経過時間                 #hud-time                左パネル
  累積標高 (elevationGainM) #hud-elev                左パネル
  標高 (altitudeM)         #hud-altitude            右パネル
  パワー (powerW)          #hud-power               中央上部・上段
  パワー低下率              #hud-power-penalty       中央上部・上段（補助表示）
  ケイデンス (cadenceRpm)   #hud-cadence             中央上部・上段
  トルク (torqueNm)        #hud-torque              中央上部・下段
  心拍 (heartRateBpm)      #hud-hr                  中央上部・上段
  勾配 (gradientPercent)   #hud-gradient            右パネル（バッジ表示）
  直近1km勾配プロファイル   #gradient-profile        左カラム下部オーバーレイ
  コース全体標高            #course-elevation-map    画面最下部（変更なし）
  3Dマップ                 #ele-view                左カラム全体（拡大）
  風景写真                 .mapillary-panel         右カラム上半分
  2Dマップ                 #map-inner               右カラム下半分
  ゴール表示                #hud-finish-msg          中央上部付近（要検討）

  ※ #hud-finish-msg（"ゴール！"表示）の新レイアウトでの配置は
    本仕様では未確定。実装時に別途決定する。

--------------------------------------------------------------------
6. レスポンシブ／ポートレート対応
--------------------------------------------------------------------
  - 本仕様は横向き（landscape）を前提とする（iPad想定）
  - 縦向き（portrait）時のフォールバック仕様は既存の
    @media (orientation: portrait) ルールを踏襲しつつ、
    3Dマップ＋HUDオーバーレイを優先表示する形で別途検討する
    （本仕様の対象外、次フェーズで詳細化）
